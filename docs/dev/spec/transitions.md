# ページ遷移・インタラクション仕様 (Transitions)

## 1. 概要
本プロジェクトにおける画面遷移は、単なる情報の切り替えではなく、ユーザーに「空間的な一貫性」を感じさせる体験（UX）として定義する。
すべての遷移は、進入した経路を辿って戻る「逆再生（Mirroring）」を原則とする。

## 2. 遷移プロトコル

### 2.1. パターンA：水平スライド (Horizontal Slide)
**用途:** 階層の移動（一覧 ⇄ 詳細）など、一般的なページ遷移。

- **進入 (Forward/Push):** 
  - `initial`: `x: "100%", opacity: 0`
  - `animate`: `x: 0, opacity: 1`
- **退出 (Backward/Pop - 逆再生):**
  - `exit`: `x: "100%", opacity: 0`
- **設計指針:** 右から左へ「進む」ことで探索の深化を表現し、右へ「戻る」ことで元の場所への復帰を表現する。

### 2.2. パターンB：ヒーロー遷移 (Hero / Shared Element)
**用途:** 予告編再生ボタンのクリック、特定のカード要素からの拡大表示。

- **技術実装:** `framer-motion` の `layoutId` を使用。
- **進入:** クリック対象の要素（ボタン等）の形状・位置から、プレイヤーの枠組みへと滑らかに拡大。
- **退出 (逆再生):** プレイヤーが元のボタンのサイズ・位置へと収束しながら消滅。
- **設計指針:** 視線の連続性を維持し、UIが「バラバラの部品」ではなく「一つの生命体」のように動く演出を行う。

### 2.3. パターンC：垂直スライド (Vertical Slide / Drawer-like)
**用途:** ホーム/一覧ページへの遷移、またはボトムアップでせり上がる特定のセクション。

- **進入 (Forward):** 
  - `initial`: `y: "100%", opacity: 0.8`
  - `animate`: `y: 0, opacity: 1`
- **退出 (Backward - 逆再生):**
  - `exit`: `y: "100%", opacity: 0`
- **ウザくない挙動への配慮:** 
  - 単純な直線移動ではなく、後述の `Ease-Out` を適用。
  - 進入時の `initial` で `opacity` を 0.8 程度に設定することで、背後からスッと現れる透明感のある出現を演出。

## 3. モーション・ガイドライン

### 3.1. イージング (Easing)
「滑らかさ」の鍵として、以下の数値を推奨する。
- **Curve:** `[0.22, 1, 0.36, 1]` (Cubic Bezier - Out-Quint)
  - 最初は速く、目的地で優雅に減速する。
- **Duration:** 
  - 水平/垂直スライド: 0.4s 〜 0.5s
  - ヒーロー遷移: 0.35s

### 3.2. 逆再生の判定ロジック
React Router の `useNavigationType` または `location.state` を監視し、方向を判定する。
```typescript
const isBack = navigationType === 'POP';
const customX = isBack ? "-100%" : "100%"; // 逆再生の計算ロジック
```

## 4. 特記事項
- **アクセシビリティ:** `prefers-reduced-motion` 設定が有効なユーザーに対しては、スライドをスキップし、シンプルなフェードインのみに切り替えるフォールバックを用意すること。
