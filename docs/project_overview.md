# プロジェクト概要

## アプリケーション
- **種別**: 映画情報統合プラットフォーム (BFF 構成)
- **目的**: 映画作品を軸に、予告編、ニュース、配信状況、考察記事を一箇所に集約し、情報の分断を解消する。
- **規模**: 個人/小規模チーム向け (Vercel での公開を想定)
- **ドメイン**: 映画メタデータ、動画コンテンツ、ニューススクレイピング、Web 検索

## 技術スタック
- **フロントエンド**: React 19 (Vite), TypeScript, React Router 7, TanStack Query v5, Tailwind CSS, Framer Motion
- **バックエンド**: Node.js (Express 5), TypeScript
- **DB/Cache**: NodeCache (インメモリ), Upstash Redis (分散キャッシュ) ※RDB/NoSQL は現状未導入
- **インフラ**: Vercel (Frontend/API), SerpApi, TMDB API, YouTube API

## 現状の非機能要件
- **パフォーマンス**: 外部 API レートリミット回避と高速化のため、インフラ層での多重キャッシュ戦略を採用。
- **セキュリティー**: 環境変数による API キー管理。ユーザー認証は未実装（今後の展望）。
- **可用性**: サーバーレス環境（Vercel）へのデプロイを前提としたステートレス設計。
- **保守性**:
    - **Backend**: Clean Architecture による外部依存の隔離とテスト容易性の確保。
    - **Frontend**: Feature-Sliced Design (FSD) 的な機能分割。
    - **Shared**: Monorepo による型定義の完全同期。

## 課題と背景
- **背景**: 映画に関する情報の分断（ニュース、予告、考察が別サイトに散らばっている）を解決するためのプロトタイプ開発。
- **既知の課題**: ユーザー認証が未実装であるため、ウォッチリスト等のパーソナライズ機能がローカルストレージ依存となっている。
